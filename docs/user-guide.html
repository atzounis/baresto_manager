Έναρξη & σύνδεση

Getting started & login

Ανοίξτε την εφαρμογή στον browser του υπολογιστή ή του κινητού (π.χ. http://127.0.0.1:8765 στον τοπικό server). Η θύρα προεπιλογής είναι 8765, όχι 8000.

Open the app in your computer or phone browser (e.g. http://127.0.0.1:8765 on a local server). The default port is 8765, not 8000.

4-digit PIN on a large keypad — ideal for a shared waiter tablet. A successful login starts the active shift.

Windows — γρήγορη εκκίνηση

Windows — quick start

Για PC με Windows 10/11 χωρίς Command Prompt: κατεβάστε το ZIP από GitHub, εξαγάγετέ το και κάντε διπλό κλικ στο Start-BarestoManager.bat (δίπλα στο manage.py).

On Windows 10/11 without using Command Prompt: download the GitHub ZIP, extract it, and double-click Start-BarestoManager.bat (next to manage.py).

  1. Την πρώτη φορά, ο launcher εγκαθιστά Python (μέσω winget αν λείπει), βιβλιοθήκες, .env και migrations — μπορεί να πάρει 5–15 λεπτά.
  2. On first run, the launcher installs Python (via winget if missing), libraries, .env, and migrations — may take 5–15 minutes.
  3. Ανοίγει αυτόματα ο browser στο /login/. Αφήστε το παράθυρο κονσόλας ανοιχτό· Ctrl+C για διακοπή.
  4. Your browser opens /login/ automatically. Leave the console window open; press Ctrl+C to stop.
  5. Για κινητά στο ίδιο Wi‑Fi, ο launcher προσθέτει κανόνα firewall για θύρα 8765. Αν το κινητό δεν συνδέεται, τρέξτε ως διαχειριστής το scripts\windows\Add-Firewall-Rule.bat.
  6. For phones on the same Wi‑Fi, the launcher adds a firewall rule for port 8765. If the phone still cannot connect, run scripts\windows\Add-Firewall-Rule.bat as administrator.
  7. Προαιρετικά: χτίστε BarestoManager.exe με scripts\windows\build-exe.bat (μόνο σε Windows).
  8. Optional: build BarestoManager.exe with scripts\windows\build-exe.bat (Windows PC only).

Ρόλοι & δικαιώματα

Roles & permissions

Μετά τη σύνδεση, η αρχική σελίδα εξαρτάται από τον ρόλο: σερβιτόρος → τραπέζια, κουζίνα → οθόνη κουζίνας, λοιποί → πίνακας ελέγχου.

After login, the home page depends on role: waiter → tables, kitchen → kitchen display, others → dashboard.

Waiter ΣερβιτόροςWaiter

Τραπέζια, παραγγελίες, αποστολή στην κουζίνα, παράδοση έτοιμων πιάτων, κλείσιμο τραπεζιού (με επιλογή πληρωμής), ιστορικό παραγγελιών.

Tables, orders, send to kitchen, mark ready items delivered, close table (with payment selection), order history.

Cashier ΤαμείοCashier

Ίδια πλοήγηση με σερβιτόρο (τραπέζια, παραγγελίες). Επιπλέον δικαιώματα για έκπτωση και κλείσιμο λογαριασμού (όπου υλοποιηθεί στη ροή).

Same navigation as waiter (tables, orders). Extra permissions for discounts and closing bills (where implemented in the flow).

Kitchen ΚουζίναKitchen

Μόνο η οθόνη κουζίνας (KDS): λήψη παραγγελιών, ενημέρωση κατάστασης ανά πιάτο, φίλτρο σταθμού (κουζίνα / bar / grill).

Kitchen display (KDS) only: receive orders, update item status, filter by station (kitchen / bar / grill).

Manager ΔιαχειριστήςManager

Μενού (δίγλωσσο), εταιρεία / νομικά στοιχεία, QR επισκεπτών, τραπέζια & όροφοι, κάτοψη, αναφορές, κουζίνα.

Bilingual menu, company details, guest QR, tables & floors, floor plan, reports, kitchen access.

Admin Διαχ. συστήματοςAdministrator

Ό,τι ο διαχειριστής, συν διαχείριση χρηστών προσωπικού (/staff/users/).

Everything a manager can do, plus staff user management (/staff/users/).

Σερβιτόρος / ταμείο — ροή εργασίας

Waiter / cashier — workflow

1. Τραπέζια (/tables/)

1. Tables (/tables/)

Δείτε λίστα ή κάτοψη όρου. Φιλτράρετε ανά όροφο. Χρώματα κατάστασης:

View list or floor plan. Filter by floor. Status colours:

Ελεύθερο
Free
Κατειλημμένο
Occupied
Έτοιμα πιάτα (badge)
Ready items (badge)

Το badge «έτοιμα» δείχνει πόσα πιάτα είναι έτοιμα από την κουζίνα αλλά δεν έχουν παραδοθεί ακόμα. Στην κεφαλίδα παραγγελίας εμφανίζεται ένα badge (όχι διπλότυπο).

The ready badge shows how many kitchen-ready items have not been delivered yet. On the order screen, a single badge appears in the header.

Ειδοποιήσεις στην οθόνη τραπεζιών

Alerts on the tables screen

  • Κλήση σερβιτόρου: όταν πελάτης πατά «Καλέστε σερβιτόρο» στο QR μενού, εμφανίζεται modal με τραπέζι/όροφο (ήχος + ειδοποίηση).
  • Τραπέζι έκλεισε: όταν άλλος σερβιτόρος κλείνει τραπέζι, η οθόνη /tables/ ενημερώνεται live (modal «Το τραπέζι έκλεισε», επιλογή εκτύπωσης απόδειξης).
  • Σε κινητό: πατήστε την μπάρα Tap to enable sound & alerts μία φορά για ήχο και ειδοποιήσεις.
  • Guest calls waiter: when a customer taps Call waiter on the QR menu, a modal shows table/floor (sound + notification).
  • Table closed: when another device closes a table, /tables/ updates live (modal “Table is closed”, option to print receipt).
  • On mobile: tap Tap to enable sound & alerts once for sound and notifications.

2. Άνοιγμα τραπεζιού & παραγγελία

2. Open table & order

  1. Πατήστε Άνοιγμα τραπεζιού σε ελεύθερο τραπέζι — ορίστε αριθμό καλυψών (ατόμων).
  2. Tap Open table on a free table — set cover count (number of guests).
  3. Ανοίγει η οθόνη παραγγελίας (/orders/new/<session>/) — βελτιστοποιημένη για κινητό.
  4. The ordering screen opens (/orders/new/<session>/) — optimised for mobile.
  5. Πατήστε κατηγορία → πιάτο για προσθήκη. Το συρτάρι παραγγελίας (κάτω) δείχνει live το σύνολο.
  6. Tap category → dish to add. The order drawer (bottom) shows the live total.
  7. Προαιρετικά: σημειώσεις ανά πιάτο, modifiers αν υπάρχουν.
  8. Optional: notes per item, modifiers where configured.
  9. Σε κινητό, η οθόνη παραγγελίας χρησιμοποιεί μεγαλύτερα γράμματα και κουμπιά για ευκολότερη χρήση με το δάχτυλο.
  10. On mobile, the order screen uses larger text and buttons for easier one-handed use.

3. Αποστολή στην κουζίνα

3. Send to kitchen

  1. Όταν υπάρχουν νέα πιάτα (δεν έχουν σταλεί ακόμα), πατήστε Αποστολή στην κουζίνα στο συρτάρι παραγγελίας (κάτω).
  2. When there are new items (not yet sent), tap Send to kitchen in the order drawer (bottom).
  3. Ανοίγει πλήρους οθόνης βήμα: προαιρετικά γράψτε Σημείωση κουζίνας (π.χ. αλλεργία, χωρίς πάγο, σερβίρετε μαζί).
  4. A full-screen step opens: optionally enter a Kitchen note (e.g. allergy, no ice, serve together).
  5. Πατήστε Αποστολή παραγγελίας για αποστολή ή Ακύρωση για επιστροφή χωρίς αποστολή.
  6. Tap Send order to send, or Cancel to go back without sending.
  7. Τα νέα πιάτα περνούν στην κουζίνα· η οθόνη KDS ενημερώνεται σε πραγματικό χρόνο. Η σημείωση εμφανίζεται στην κουζίνα στο ticket και στο modal νέας παραγγελίας.
  8. New items go to the kitchen; the KDS updates in real time. The note appears on kitchen tickets and in the new-order alert modal.
  9. Μπορείτε να συνεχίσετε να προσθέτετε πιάτα και να στέλνετε ξανά (κάθε αποστολή μπορεί να έχει δική της σημείωση).
  10. You can keep adding items and sending again (each send can include its own note).

4. Έτοιμα πιάτα & ειδοποιήσεις

4. Ready items & alerts

Όταν η κουζίνα σημάνει πιάτο ως έτοιμο, λαμβάνετε ειδοποίηση (ήχος / websocket). Στην παραγγελία ή στα τραπέζια, σημειώστε το πιάτο ως παραδομένο όταν το σερβίρετε.

When the kitchen marks an item ready, you get an alert (sound / WebSocket). On the order or tables screen, mark the item delivered when served.

5. Κλείσιμο τραπεζιού, πληρωμή & απόδειξη

5. Close table, payment & receipt

  1. Πατήστε Κλείσιμο τραπεζιού όταν οι πελάτες φύγουν.
  2. Tap Close table when guests leave.
  3. Επιλέξτε Πληρωμή σε πλήρους οθόνης παράθυρο: Μετρητά ή Πιστωτική κάρτα, μετά Επιβεβαίωση και κλείσιμο (ή Ακύρωση για επιστροφή).
  4. Choose Payment in a full-screen step: Cash or Credit card, then Confirm and close (or Cancel to go back).
  5. Το σύστημα δεν επιτρέπει κλείσιμο αν: υπάρχουν μη αποσταλμένα πιάτα, πιάτα ακόμα στην κουζίνα, ή έτοιμα πιάτα που δεν παραδόθηκαν.
  6. The system blocks closing if: unsent items remain, items are still in the kitchen, or ready items were not delivered.
  7. Μετά το κλείσιμο: δημιουργείται λογαριασμός με τον τρόπο πληρωμής, το τραπέζι γίνεται ελεύθερο, ανακατευθύνεστε στο /tables/.
  8. After closing: a bill is created with the payment method, the table becomes free, you are redirected to /tables/.
  9. Εμφανίζεται μήνυμα «Το τραπέζι έκλεισε» και modal με τρόπο πληρωμής (έντονα), Εκτύπωση απόδειξης, Όχι τώρα ή κουμπί κλεισίματος (×). Η απόδειξη δείχνει τον τρόπο πληρωμής.
  10. A “Table is closed” message and modal appear with the payment method (bold), Print receipt, Not now, or close (×). The receipt shows the payment method.
  11. Άλλες συσκευές στην /tables/ (π.χ. PC ταμείου) λαμβάνουν την ίδια ειδοποίηση σε πραγματικό χρόνο.
  12. Other devices on /tables/ (e.g. cashier PC) receive the same alert in real time.
Ιστορικό / επανεκτύπωση: /orders/ — φιλτράρετε ανά ημερομηνία και ξανατυπώνετε απόδειξη. CSV για διαχειριστές.
History / reprint: /orders/ — filter by date and reprint receipts. CSV export for managers.

Sign in as kitchen or use PIN 3333. You land on /kitchen/ automatically.

Χρήση της οθόνης

Using the display

  • Οι παραγγελίες εμφανίζονται ως tickets ανά τραπέζι, με νεότερες πρώτες.
  • Φιλτράρετε ανά σταθμό: κουζίνα, bar, grill.
  • Για κάθε πιάτο: Επόμενο / Advance — κατάσταση: αναμονή → προετοιμασία → έτοιμο → σερβιρισμένο.
  • Αν ο σερβιτόρος έγραψε σημείωση κουζίνας, εμφανίζεται έντονα στο ticket και στο modal νέας παραγγελίας.
  • Νέες παραγγελίες: ηχητική ειδοποίηση (αν ενεργή στο browser).
  • Οι ενημερώσεις συγχρονίζονται live (WebSocket) σε όλες τις οθόνες κουζίνας.
  • Orders appear as tickets per table, newest first.
  • Filter by station: kitchen, bar, grill.
  • Per item: Advance — status flow: pending → preparing → ready → served.
  • If the waiter entered a kitchen note, it appears prominently on the ticket and in the new-order alert modal.
  • New orders trigger a sound alert (if enabled in the browser).
  • Updates sync live (WebSocket) across all kitchen screens.
Κρατήστε τον browser σε πλήρη οθόνη (F11) σε dedicated οθόνη κουζίνας. Μην κλείνετε το tab — η ουρά εξαφανίζεται μόνο όταν όλα τα πιάτα σημειωθούν σερβιρισμένα ή η παραγγελία πληρωθεί.
Use full-screen browser (F11) on a dedicated kitchen monitor. Keep the tab open — tickets clear when all items are served or the order is paid.

Νέα παραγγελία — modal ειδοποίησης

New order — alert modal

Όταν ο σερβιτόρος στέλνει νέα πιάτα, εμφανίζεται modal «Νέα παραγγελία από σερβιτόρο» με τραπέζι, όροφο, λίστα πιάτων και προαιρετική σημείωση κουζίνας (ήχος + ειδοποίηση). Πατήστε OK ή κλείστε (×) για να συνεχίσετε. Σε κινητό/tablet: πατήστε μία φορά Tap to enable sound & alerts.

When a waiter sends new items, a “New order from waiter” modal shows table, floor, item list, and an optional kitchen note (sound + notification). Tap OK or close (×) to continue. On mobile/tablet: tap Tap to enable sound & alerts once.

Συμβουλές & αντιμετώπιση προβλημάτων

Tips & troubleshooting

  • Γλώσσα διεπαφής: ΕΛ / EN στη γραμμή πλοήγησης — αλλάζει labels, όχι τα δεδομένα μενού (αυτά επεξεργάζονται στο /menu/).
  • Real-time: Τοπικά χρησιμοποιείται in-memory WebSocket· αν η κουζίνα δεν ενημερώνεται, ανανεώστε τη σελίδα ή ελέγξτε ότι ο server τρέχει.
  • Κινητό δεν συνδέεται: Ίδιο Wi‑Fi, firewall, σωστή IP και θύρα 8765.
  • QR δεν ανοίγει στο κινητό: Ελέγξτε SITE_BASE_URL και τρέξτε python manage.py generate_qr_codes.
  • Ήχος & ειδοποιήσεις: Σε κινητό/tablet, πατήστε Tap to enable sound & alerts μία φορά στην /tables/ ή /kitchen/.
  • Αποστολή στην κουζίνα: Μετά την προσθήκη πιάτων, πατήστε Αποστολή στην κουζίνα — δεν χρειάζεται ανανέωση σελίδας. Αν δεν βλέπετε το κουμπί Αποστολή παραγγελίας, κάντε scroll στο modal ή ακυρώστε και ξανανοίξτε το βήμα.
  • Κλείσιμο τραπεζιού: Επιλέξτε τρόπο πληρωμής (μετρητά/κάρτα) πριν την επιβεβαίωση· η απόδειξη εμφανίζει τον τρόπο πληρωμής.
  • Δεν κλείνει το τραπέζι: Στείλτε όλα τα πιάτα, περιμένετε την κουζίνα, παραδώστε τα έτοιμα — μετά δοκιμάστε ξανά.
  • Αποσύνδεση: Από το μενού προφίλ — τερματίζει και την ενεργή βάρδια.
  • UI language: ΕΛ / EN in the nav bar — changes labels, not menu data (edit those at /menu/).
  • Real-time: Local dev uses in-memory WebSockets; if KDS stalls, refresh or confirm the server is running.
  • Phone won’t connect: Same Wi‑Fi, firewall, correct IP and port 8765.
  • QR won’t open on phone: Check SITE_BASE_URL and run python manage.py generate_qr_codes.
  • Sound & alerts: On mobile/tablet, tap Tap to enable sound & alerts once on /tables/ or /kitchen/.
  • Send to kitchen: After adding items, tap Send to kitchen — no page refresh needed. If you don’t see Send order, scroll within the modal or cancel and open the step again.
  • Close table: Choose payment method (cash/card) before confirming; the receipt shows how the bill was paid.
  • Can’t close table: Send all items, wait for kitchen, deliver ready items — then retry.
  • Logout: From the profile menu — also ends the active shift.
